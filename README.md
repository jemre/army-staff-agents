# Army Staff Agents

A repository for prompt engineering, context management, and agent definitions that enable LLMs to act as Army Staff officers. Also contains materials for a professional development class on Prompt and Context Engineering.

---

## Purpose

1. **Army Staff Agents** — Define, configure, and run LLM agents that embody Army staff roles. Agents can be spun up individually or coordinated through an Executive Officer (XO) agent that delegates tasks and synthesizes staff products.
2. **Professional Development (PD) Class** — Store all materials related to the Prompt and Context Engineering instruction, including slides, exercises, and reference examples.

## How It Works

### Spinning Up an Agent

Each folder under `agents/` contains the definition for that staff role:
- **system prompt** — establishes role identity, responsibilities, and behavior
- **context** — relevant unit, mission, and SOP documents to inject
- **skills/tools** — any specialized capabilities the agent has access to

### XO-Led Coordination

The recommended starting point is the **XO agent** (`agents/xo/`). The XO:
- Receives a task or commander's intent
- Breaks it down and delegates to the appropriate staff sections
- Collects and synthesizes outputs into a final staff product

### Staff Products

Completed outputs (briefings, OPORDs, risk assessments, staff estimates, etc.) are saved in `products/`, organized by date and type.
