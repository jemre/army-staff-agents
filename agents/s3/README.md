# S3 Agent — Operations Officer

The S3 is the primary advisor on all operations and training matters. Leads the MDMP, produces all operational orders, manages the training schedule, and synchronizes the battle rhythm.

## Files

| File | Purpose |
|------|---------|
| `system_prompt.md` | Core S3 prompt — LLM-agnostic, works in Claude, GPT, Gemini, etc. |

## Quick Start (Claude Code)

```
/s3
```

## Quick Start (Browser / Other LLMs)

Copy the contents of `system_prompt.md` into the system prompt field.

## Key Products

| Product | Format |
|---------|--------|
| OPORD | 5-paragraph (Situation, Mission, Execution, Sustainment, C2) |
| FRAGO | Changes only; references base OPORD |
| WARNO | Early warning with initial guidance |
| Training Schedule | Weekly and long-range (12–18 month) |
| COA Brief | Statement, sketch, advantages/disadvantages, recommendation |
| Execution Matrix | Timeline of events synchronized across units |
| Synchronization Matrix | Warfighting functions across phases/time |
| Risk Assessment | DD Form 2977 format |

## MDMP at a Glance

```
Receipt of Mission → Mission Analysis → COA Development
→ COA Analysis (War Game) → COA Comparison → COA Approval → Orders Production
```

## Coordination With Other Sections

The S3 is the central synchronizer. Key dependencies:
- **S2** — Enemy situation, IPB, PIR
- **S4** — Logistics and CSS plan
- **S1** — Strength and personnel replacement
- **S6** — Signal annex and frequencies
- **FSO** — Fire support plan
