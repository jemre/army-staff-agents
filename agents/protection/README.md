# Protection Agent — Protection WfF Specialist

Functional resource advising the S3 on all Protection Warfighting Function matters. At battalion/brigade, the S3 is the de facto Chief of Protection — this agent supports that role. Outputs flow through S3, not directly to XO.

## Quick Start (Claude Code)

```
/protection
```

## Doctrinal Position

- **WfF:** Protection (preserve the force)
- **Staff type:** Protection special staff functions aggregated; coordinated through S3
- **No dedicated Chief of Protection at battalion/brigade** — S3 owns this WfF
- **Coordinates:** CBRN, EOD, OPSEC, Safety, Provost Marshal, Personnel Recovery, AMD, Physical Security

## Key Products

| Product | Purpose |
|---------|---------|
| Protection Annex (Annex E) | Force protection plan integrated into OPORD |
| OPSEC Plan | EEFI list, vulnerability assessment, countermeasures |
| CBRN Assessment and Plan | Detection, avoidance, decontamination |
| Force Protection Plan | Physical security, TCPs, QRF triggers |
| Risk Assessment (DD 2977) | Consolidated protection risk across all functions |
| Protection Working Group Agenda | Recurring WfF synchronization |

## Coordination Chain

Protection → S3 → XO (not Protection → XO directly)

## Important Note on Threat Data

Protection does not generate its own threat assessments. All threat input comes from the S2. If S2 products are unavailable, this must be flagged as a gap.
